# v0.9.5 plain digest-decision audit

Date: 2026-07-15. Parent: v0.9.4.

## Customer outcome

When the morning digest asks what to do with a stale task, the owner can type
`keep`, `tomorrow`, or `drop` as a normal message. A stale waiting prompt also
accepts plain `back on`. Telegram's reply gesture remains supported but is no
longer required to make Hob understand the exact choices it advertises.

## Root cause and production evidence

The live digest asked the owner to “Reply keep, tomorrow, or drop.” The inbound
`Keep` update had `reply_to: null`, which is normal when someone simply types in
the chat box. Hob only looked up an item anchor through `_replied_item`, so the
turn reached the model without a subject and safely degraded to “I did not catch
a task there.” The same pinned digest already had an internal reference to the
correct stale item; the edge failed to use it for a plain response.

## Deterministic evidence

1. Plain `Keep` with no reply metadata updates the prompted item and never calls
   the model.
2. Plain `tomorrow` moves exactly that item to the next local date.
3. Plain `drop` drops exactly that item.
4. Plain `back on` resumes exactly the waiting item.
5. A consumed or previous-day prompt cannot act again.
6. Newer conversational task focus prevents terse `tomorrow` or `drop` from
   being captured by the older digest prompt.
7. A digest sent before this upgrade can use its existing pinned-message
   reference for the rest of that day.
8. A digest with no actionable nudge clears prior decision context.

## Remaining risks

1. A same-day prompt intentionally remains available through unrelated
   non-mutating chat. This matches the outstanding question but should be
   revisited if daily use shows surprising late responses.
2. Only exact short decision phrases bypass the model. Longer explanations
   continue through normal interpretation and reference guards.
3. The Store edition must carry this typed proactive-context contract when its
   Telegram transport is connected; its current background runtime remains
   intentionally locked.

## Decision

Release as v0.9.5 after Python, Swift, lock, syntax, signed EventKit, Xcode,
real-model, and exact-commit Ubuntu/macOS CI gates pass. Back up the live
database, deploy by graceful restart, apply the missed production `keep`
through Hob's normal undoable action path, and verify clean queues.
