# v0.9.2 displayed-list scope audit

Date: 2026-07-11. Trigger: live phone screenshot after an evening plan check.

## Finding

A reply equivalent to “move everything on that list to Monday except one item,
which goes Sunday” correctly handled the exception dates but also moved open
future tasks that had not appeared in the evening message. The model emitted a
broad bulk scope, and deterministic reconciliation expanded it over the full
open set because Hob did not persist the exact proactive list it had displayed.

## Customer and usability impact

This violated conversational reference expectations and could silently disturb
unrelated plans. The reply looked internally consistent, so a customer had to
notice extra lines and repair them manually. This is a correctness defect, not
merely awkward wording.

## Resolution

- After successful evening delivery, Hob persists the exact actionable item ids
  and labels shown, including adopted-plan items and any “also on deck” items.
- The presented list expires after 24 hours and filters out tasks that are no
  longer open.
- “That list”, “this list”, “those tasks/items/ones”, and equivalent bulk turns
  are intersected with the presented ids regardless of the model's scope.
- If no current presented list exists, Hob asks which list and changes nothing.
- The model prompt receives the same presented order for accurate references,
  while the deterministic intersection remains the safety boundary.

| Accessibility | Plain-text list replies remain sufficient; no button or visual-only state is introduced | No regression. |

## Evidence

- Planner regression covers a three-item presented list plus two unrelated open
  tasks and a different-date exception.
- MessageService integration proves only the presented ids change in storage.
- EOD regression proves the persisted context matches the actionable displayed
  list and excludes canceled or missing plan references.
- The complete real-model corpus passes 74/74 with the natural list instruction.
